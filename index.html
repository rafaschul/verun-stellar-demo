Security

Sybil Resistance

Four independent defense layers prevent identity spoofing, replay attacks, and unauthorized agent access.

🔢

Soroban Auth Nonce

Each InvokeHostFunctionOp includes a unique nonce embedded in the authorization entry. Replay attacks are cryptographically blocked at the Soroban layer.

⏱

max_ledger Expiry

Auth entries include a max_ledger bound. Stale or replayed entries are rejected as expired — no grace window for attackers.

🏛

2-of-3 Validator Consensus

Three independent validators must each approve a ATS score. No single validator can grant or revoke access — eliminating single points of failure.

🔍

On-chain Audit Trail

Every evaluation verdict, score update, and fee distribution is written to Stellar via Soroban contracts. Immutable, queryable, verifiable by anyone.
